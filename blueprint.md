# 학생 이수 학점 점검 도구 - 블루프린트

## 1. 프로젝트 개요

본 프로젝트는 사용자가 엑셀(Excel) 파일을 업로드하면, 학생들의 교과목별 이수 학점을 자동으로 분석하고 필수 이수 학점 충족 여부를 검사하여 결과를 보여주는 웹 애플리케이션을 개발하는 것을 목표로 한다. 복잡하고 비정형적인 데이터 구조를 가진 엑셀 문서를 효과적으로 파싱하고, 사용자가 직접 필수 이수 학점 기준을 수정할 수 있는 유연한 인터페이스를 제공한다.

## 2. 핵심 기능

- **엑셀 파일 업로드**: 사용자는 `.xlsx` 또는 `.xls` 형식의 학생 성적 데이터 파일을 업로드할 수 있다.
- **필수 이수 학점 기준 관리**:
    - 국어, 수학, 영어, 한국사, 사회, 과학, 생활/교양 등 각 교과 영역의 필수 이수 학점 기준을 화면에 표시한다.
    - 사용자는 각 교과의 학점 기준을 직접 입력하고 수정할 수 있다.
- **데이터 파싱 및 분석**:
    - 업로드된 엑셀 데이터에서 학생 정보, 학년, 학기, 교과, 과목명, 이수 학점을 추출한다.
    - 비정형적인 데이터(예: 병합된 셀, 여러 줄로 나뉜 학생 정보)를 효과적으로 처리한다.
    - '한국사' 과목을 '사회'가 아닌 '한국사' 교과로 재분류한다.
    - '기술·가정/정보/제2외국어/교양' 과목들을 '생활/교양' 교과 영역으로 통합하여 계산한다.
- **결과 표시**:
    - 학생별로 각 필수 교과 영역의 이수 학점과 충족 여부를 명확하게 표시한다.
    - 미충족 시, 부족한 학점을 계산하여 함께 보여준다.
    - 전체적인 디자인은 깔끔하고 직관적으로 구성하여 사용자의 가독성을 높인다.

## 3. 기술 스택 및 디자인

- **Frontend**: HTML, CSS, JavaScript (ES Modules)
- **라이브러리**: SheetJS (XLSX) - 엑셀 파일 파싱을 위해 CDN 방식으로 추가
- **디자인**:
    - 모던하고 깔끔한 UI/UX
    - 반응형 웹 디자인을 적용하여 다양한 디바이스에서 최적의 경험을 제공
    - 직관적인 아이콘과 색상 체계를 사용하여 정보의 가독성을 높임
    - 동적인 효과와 부드러운 애니메이션으로 사용자 경험 향상

## 4. 개발 계획 (현재 단계)

### **Phase 1: UI/UX 설계 및 기본 구조 생성 (진행 중)**

1.  **`blueprint.md` 생성 (완료)**: 프로젝트의 목적, 기능, 개발 계획을 명확히 정의.
2.  **`index.html` 구조 설계**:
    - 페이지 제목 및 설명 영역 추가.
    - 필수 이수 학점 기준을 관리할 수 있는 테이블 UI 생성. 각 교과목과 학점 입력 필드를 포함.
    - 파일 업로드를 위한 `<input type="file">` 요소 및 분석 실행 버튼 추가.
    - 분석 결과를 동적으로 표시할 영역 확보.
    - SheetJS 라이브러리 CDN 추가.
3.  **`style.css` 디자인 적용**:
    - 전체적인 레이아웃, 색상, 폰트 등 시각적 요소 스타일링.
    - 사용성과 심미성을 고려한 모던 디자인 적용 (입력 필드, 버튼, 결과 테이블 등).
4.  **`main.js` 초기 로직 구현**:
    - 파일 업로드 이벤트 리스너 추가.
    - 엑셀 파일을 읽고 JSON 데이터로 변환하는 기본 로직 구현.
    - 필수 이수 학점 기준을 관리하는 데이터 구조 정의.

### **Phase 2: 데이터 분석 및 결과 처리**

1.  **데이터 파싱 로직 고도화**: 제공된 엑셀 샘플의 복잡한 구조(학생 정보, 과목 정보)를 정확하게 파싱하는 함수 개발.
2.  **교과 분류 로직 구현**: '한국사' 및 '생활/교양' 과목 분류 규칙 적용.
3.  **학점 계산 로직 구현**: 학생별로 교과 영역에 따른 총 이수 학점 계산.
4.  **결과 비교 및 생성**: 계산된 이수 학점과 필수 기준을 비교하여 결과(충족/미충족/부족 학점) 생성.
5.  **결과 화면 출력**: 분석 결과를 HTML 테이블 형태로 동적으로 생성하여 화면에 렌더링.

### **Phase 3: 테스트 및 고도화**

1.  **다양한 예외 케이스 테스트**: 여러 형식의 엑셀 파일 및 데이터로 테스트 진행.
2.  **오류 처리**: 파일 형식이 다르거나 분석이 불가능한 데이터에 대한 예외 처리 로직 추가.
3.  **사용자 피드백 반영 및 개선**: 추가적인 요구사항이나 개선점을 반영하여 최종 완성도 향상.
